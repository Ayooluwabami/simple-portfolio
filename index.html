How the stack works

Every request from the internet hits Lightsail — never EC2 directly. Static assets are served from Lightsail's SSD. Dynamic requests are forwarded privately to EC2 via the peered VPC connection.

  1. 01 User's browser sends a request to the Lightsail public IP on port 80 or 443.
  2. 02 Nginx on Lightsail checks the request — static file? Served directly from local SSD.
  3. 03 Dynamic request? Nginx proxies it to the EC2 private IP via the VPC peering link.
  4. 04 EC2 processes the request. Apache returns the response to Lightsail.
  5. 05 Lightsail sends the final response to the user. All outbound bandwidth charged at the cheap bundled rate.
Key Insight EC2 is invisible to the public internet. Its security group only accepts inbound traffic from the Lightsail VPC CIDR — not from 0.0.0.0/0. That's the security win of this pattern.
